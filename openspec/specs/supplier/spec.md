## Purpose
供应商管理：支持供应商的分页查询、搜索、新增、编辑、禁用、查看详情等基础操作。

## Fields
| 字段名           | 中文名     | 类型     | 约束/说明                         |
|----------------|---------|---------|----------------------------------|
| id             | 供应商ID  | int     | 主键，自增                        |
| name           | 供应商名称 | string  | 必填，1-200 字符                  |
| contact_person | 联系人    | string  | 可选，1-100 字符                   |
| contact_phone  | 联系电话  | string  | 可选，1-20 字符                    |
| status         | 状态      | string  | 必填，枚举：enabled/disabled，默认 enabled |
| remark         | 备注      | string  | 可选，最长 128 字符                |
| created_by     | 创建者ID  | int     | 外键，关联 users.id               |
| created_at     | 创建时间  | string  | 创建时间，ISO8601                  |
| updated_at     | 更新时间  | string  | 更新时间，ISO8601                  |

## Requirements


### Requirement: 供应商列表查询
系统 SHALL 支持供应商列表的展示，支持按供应商名称模糊搜索、按状态筛选。

#### Scenario: 加载供应商列表
- **WHEN** 用户进入供应商管理页面
- **THEN** 系统自动加载供应商数据

#### Scenario: 输入关键词搜索
- **WHEN** 用户在搜索框输入关键词并提交
- **THEN** 系统按关键词过滤供应商列表，显示匹配结果

#### Scenario: 按状态筛选
- **WHEN** 用户选择状态筛选条件
- **THEN** 系统按状态过滤供应商列表


### Requirement: 新增/编辑供应商
系统 SHALL 支持通过表单新增供应商和编辑供应商信息，表单字段包括：名称、联系人、联系电话、备注、状态。

#### Scenario: 新建供应商表单校验
- **WHEN** 用户点击“新增供应商”并填写表单后提交
- **THEN** 系统校验必填项（名称、状态），校验通过则保存新供应商，失败则显示错误提示

#### Scenario: 编辑供应商信息
- **WHEN** 用户在供应商列表点击“编辑”并修改信息后保存
- **THEN** 系统保存变更


### Requirement: 供应商状态变更
系统 SHALL 支持用户在供应商列表中对供应商进行禁用和启用操作。

#### Scenario: 禁用供应商
- **WHEN** 用户在供应商列表点击“禁用”按钮
- **THEN** 系统将该供应商状态设为禁用

#### Scenario: 启用供应商
- **WHEN** 用户在供应商列表点击“启用”按钮
- **THEN** 系统将该供应商状态设为启用


### Requirement: 查看供应商详情
系统 SHALL 支持查看供应商详细信息，展示所有字段。

#### Scenario: 查看详情抽屉
- **WHEN** 用户点击供应商列表的“查看”按钮
- **THEN** 系统右侧弹出抽屉，展示该供应商的全部信息

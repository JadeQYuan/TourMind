
## Purpose
订单管理：支持订单的分页查询、搜索、新增、编辑、查看、自动推进状态、字段校验、产品天数自动填充等操作。

## Fields
| 字段名         | 中文名     | 类型     | 约束/说明                         |
|--------------|----------|---------|----------------------------------|
| id           | 订单ID    | int     | 主键，自增                        |
| order_no     | 订单号    | string  | 必填，唯一，自动生成              |
| product_id   | 产品ID    | int     | 必填，关联产品                    |
| product_name | 产品名称  | string  | 冗余字段，便于展示                |
| supplier_id  | 供应商ID  | int     | 必填，关联供应商                  |
| supplier_name| 供应商名称| string  | 冗余字段，便于展示                |
| customer     | 客户姓名  | string  | 必填，1-32 字符                   |
| phone        | 联系电话  | string  | 必填，手机号格式校验               |
| days         | 天数      | int     | 必填，1-365，默认取产品天数        |
| price        | 订单金额  | number  | 必填，单位：元，精度2位            |
| status       | 状态      | string  | 必填，枚举：pending_deposit/pending_payment/completed |
| remark       | 备注      | string  | 可选，最长 128 字符                |
| created_at   | 创建时间  | string  | 创建时间，ISO8601                  |
| updated_at   | 更新时间  | string  | 更新时间，ISO8601                  |

## Requirements

### Requirement: 订单列表分页查询
系统 SHALL 支持订单列表的分页展示，默认每页 20 条。

#### Scenario: 分页加载订单列表
- **WHEN** 用户进入订单管理页面
- **THEN** 系统自动加载第一页订单数据，显示分页控件，可切换页码加载对应数据

### Requirement: 订单搜索
系统 SHALL 支持按订单号、客户姓名、产品名称等字段模糊搜索。

#### Scenario: 输入关键词搜索
- **WHEN** 用户在搜索框输入关键词并提交
- **THEN** 系统按关键词过滤订单列表，显示匹配结果

### Requirement: 订单新增/编辑
系统 SHALL 支持订单的新增和编辑，表单校验必填项，订单号自动生成且不可编辑，产品和供应商为必填项。

#### Scenario: 新增订单
- **WHEN** 用户点击“新增订单”并填写表单后提交
- **THEN** 系统校验必填项，校验通过则保存新订单，失败则显示错误提示，订单号自动生成

#### Scenario: 编辑订单
- **WHEN** 用户在订单列表点击“编辑”并修改信息后保存
- **THEN** 系统保存变更，订单号字段只读不可编辑

### Requirement: 查看订单详情
系统 SHALL 支持查看订单详细信息，展示所有字段。

#### Scenario: 查看详情抽屉
- **WHEN** 用户点击订单列表的“查看”按钮
- **THEN** 系统右侧弹出抽屉，展示该订单的全部信息

### Requirement: 订单表单产品和供应商为必填
订单新建或编辑表单保存时，系统 SHALL 校验 `product_id` 和 `supplier_id` 字段不为空；任一字段为空时 SHALL 阻止提交并显示错误提示。

#### Scenario: 未选产品禁止保存
- **WHEN** 用户在订单表单中未选择产品，点击"保存"
- **THEN** 系统显示错误提示"请选择产品"，不提交请求，抽屉保持打开

#### Scenario: 未选供应商禁止保存
- **WHEN** 用户在订单表单中未选择供应商，点击"保存"
- **THEN** 系统显示错误提示"请选择供应商"，不提交请求，抽屉保持打开

#### Scenario: 表单必填项有视觉标记
- **WHEN** 用户打开订单新建或编辑抽屉
- **THEN** 产品和供应商字段旁均显示必填标记（`required` 属性）

### Requirement: 订单表单天数自动填充
用户在订单表单中选择产品时，系统 SHALL 自动将 `form.days` 设置为所选产品的 `days` 值；用户 SHALL 可在自动填充后手动修改天数。

#### Scenario: 选择产品自动填充天数
- **WHEN** 用户在订单表单中选择某产品
- **THEN** 天数输入框的值自动变更为该产品的 `days` 字段值

#### Scenario: 天数可编辑
- **WHEN** 天数已被自动填入
- **THEN** 用户仍可直接修改天数输入框的值

#### Scenario: 清除产品不自动填充
- **WHEN** 用户清除产品选择（allow-clear）
- **THEN** 天数字段值不变，保持用户最后输入或已有的值

### Requirement: 订单状态变更
订单状态 SHALL 由系统根据收款自动推进，已完成订单不可编辑，操作栏不显示手动变更状态按钮。

#### Scenario: 操作栏无变更状态按钮
- **WHEN** 用户查看订单列表
- **THEN** 每行操作栏只显示“编辑”按钮（已完成订单为禁用状态），不显示“变更状态”下拉

#### Scenario: 已完成订单不可编辑
- **WHEN** 用户查看订单列表，某行订单状态为“已完成”
- **THEN** 该行“编辑”按钮显示为禁用（`disabled`）

#### Scenario: 非已完成订单编辑按钮可用
- **WHEN** 订单状态为“待下定”或“待付款”
- **THEN** 该行“编辑”按钮可正常点击

### Requirement: 收入账单自动推进关联订单状态
收入类账单（`bill_type === 'income'`）关联了订单并保存成功后，系统 SHALL 根据以下规则自动推进订单状态：
	- 若订单当前状态为 `pending_deposit`，SHALL 自动变更为 `pending_payment`
	- 若订单当前状态为 `pending_payment` 且该订单的累计收入账单总额 ≥ 订单价格，SHALL 自动变更为 `completed`

#### Scenario: 首笔收入账单变更为待付款
- **WHEN** 用户录入一笔收入账单并关联状态为"待下定"的订单，保存成功
- **THEN** 该订单状态自动变更为"待付款"

#### Scenario: 收齐全款变更为已完成
- **WHEN** 用户录入一笔收入账单关联状态为"待付款"的订单，保存成功，且该订单累计收入账单总额 ≥ 订单价格
- **THEN** 该订单状态自动变更为"已完成"

#### Scenario: 部分收款不变更为已完成
- **WHEN** 用户录入一笔收入账单关联状态为"待付款"的订单，保存成功，但累计收入总额 < 订单价格
- **THEN** 订单状态保持"待付款"，不自动变更

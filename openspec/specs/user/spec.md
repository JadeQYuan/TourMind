## Purpose
用户管理：支持用户的分页查询、搜索、新增、编辑、禁用、查看详情、重置密码等基础操作。

## Fields
| 字段名       | 中文名     | 类型     | 约束/说明                         |
|------------|----------|---------|----------------------------------|
| id         | 用户ID    | int     | 主键，自增                        |
| name       | 姓名      | string  | 必填，1-32 字符，用户真实姓名      |
| phone      | 手机号    | string  | 必填，唯一，手机号格式校验         |
| role       | 角色      | string  | 必填，枚举：admin/operator         |
| status     | 状态      | string  | 必填，枚举：enabled/disabled       |
| created_at | 创建时间  | string  | 创建时间，ISO8601                  |
| updated_at | 更新时间  | string  | 更新时间，ISO8601                  |
| last_login | 最后登录时间 | string  | 最后登录时间，ISO8601，可为空      |
| password   | 密码      | string  | 必填，8-32 字符，存储加密，不返回前端 |
| remark     | 备注      | string  | 备注，可选，最长 128 字符           |

## Requirements

### Requirement: 用户列表分页查询
系统 SHALL 支持用户列表的分页展示，默认每页 20 条，并根据当前登录用户的角色动态调整可见范围：
- 系统管理员可查看所有用户（包括系统管理员、管理员、助理）
- 管理员仅可查看非系统管理员用户（即管理员、助理）
并支持按手机号、角色等字段模糊搜索。

#### Scenario: 系统管理员分页加载用户列表
- **WHEN** 当前登录用户为系统管理员，进入用户管理页面
- **THEN** 系统自动加载所有用户数据，显示分页控件，可切换页码加载对应数据

#### Scenario: 管理员分页加载用户列表
- **WHEN** 当前登录用户为管理员，进入用户管理页面
- **THEN** 系统仅加载非系统管理员用户数据，显示分页控件，可切换页码加载对应数据

#### Scenario: 按工号查询
- **WHEN** 管理员在搜索框输入工号并提交
- **THEN** 系统按工号过滤用户列表，显示匹配结果

### Requirement: 新增/编辑用户
系统 SHALL 支持通过表单新增用户和编辑用户信息（如姓名、角色），手机号不可编辑。新增用户时可选角色根据当前登录用户动态调整：
- 系统管理员可添加任意角色用户（系统管理员、管理员、助理）
- 管理员仅可添加管理员和助理，不能添加系统管理员

#### Scenario: 系统管理员新建用户
- **WHEN** 当前登录用户为系统管理员，点击“新增用户”
- **THEN** 可选择添加任意角色的用户

#### Scenario: 管理员新建用户
- **WHEN** 当前登录用户为管理员，点击“新增用户”
- **THEN** 只能选择添加管理员或助理，不能添加系统管理员

#### Scenario: 编辑用户信息
- **WHEN** 管理员在用户列表点击“编辑”并修改信息后保存
- **THEN** 系统保存变更，手机号字段只读不可编辑

### Requirement: 用户状态变更
系统 SHALL 支持管理员在用户列表中对用户进行禁用和启用操作，禁用后用户无法登录，启用后可正常登录。

#### Scenario: 禁用用户
- **WHEN** 管理员在用户列表点击“禁用”按钮
- **THEN** 系统将该用户状态设为禁用，用户下次登录时被拒绝

#### Scenario: 启用用户
- **WHEN** 管理员在用户列表点击“启用”按钮
- **THEN** 系统将该用户状态设为启用，用户可正常登录

### Requirement: 查看用户详情
系统 SHALL 支持查看用户详细信息，展示所有字段。

#### Scenario: 查看详情抽屉
- **WHEN** 管理员点击用户列表的“查看”按钮
- **THEN** 系统右侧弹出抽屉，展示该用户的全部信息

### Requirement: 重置用户密码
系统 SHALL 支持管理员为用户重置密码，重置后新密码通过安全方式通知。

#### Scenario: 重置密码操作
- **WHEN** 管理员点击“重置密码”按钮并确认
- **THEN** 系统为该用户生成新密码，并通过弹窗或其他安全方式告知管理员

### Requirement: 账户安全与资料展示
系统 SHALL 支持用户在右上角信息区查看头像、姓名，并自助修改密码。

#### Scenario: 右上角用户信息正确显示姓名及样式
- **WHEN** 任意角色用户登录后查看右上角
- **THEN** 头像显示 full_name 的第一个字，背景色按角色对应颜色（system_admin 红色、admin 绿色、assistant 蓝色），悬浮卡显示完整 full_name

#### Scenario: 用户可自助修改密码
- **WHEN** 用户点击右上角“修改密码”入口，填写当前密码、新密码、确认新密码
- **THEN** 弹出 Modal，校验新密码长度≥8且包含大写、小写、数字，成功后提示并登出跳转登录页

#### Scenario: 当前密码错误
- **WHEN** 用户填写错误的当前密码，点击“确认修改”
- **THEN** 后端返回 400，前端 message.error 提示“当前密码错误”，Modal 保持打开

#### Scenario: 新密码不符合强度要求
- **WHEN** 新密码少于8位，或不包含大写、小写、数字
- **THEN** 前端提交前 message.error 提示具体原因，不发送请求

#### Scenario: 两次密码不一致
- **WHEN** 新密码与确认新密码不一致
- **THEN** 前端 message.error('两次密码不一致') 提示，不发送请求

#### Scenario: 关闭 Modal 后表单清空
- **WHEN** 用户关闭修改密码 Modal
- **THEN** 三个密码字段均被清空，下次打开为初始状态

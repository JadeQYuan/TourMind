## Purpose
资金账户管理：支持账户的分页查询、搜索、新增、编辑、停用、查看详情等基础操作。

## Fields
| 字段名      | 中文名     | 类型     | 约束/说明                         |
|------------|---------|---------|----------------------------------|
| id         | 账户ID   | int     | 主键，自增                        |
| name       | 账户名称  | string  | 必填，1-200 字符                  |
| type       | 账户类型  | string  | 必填，枚举：bank/wechat/alipay/cash/pos/other |
| user_id    | 负责人ID  | int     | 可选，外键，关联users.id          |
| remark     | 备注      | string  | 可选，最长128字符                 |
| status     | 状态      | string  | 必填，枚举：enabled/disabled，默认enabled |
| created_by | 创建者ID  | int     | 外键，关联 users.id               |
| created_at | 创建时间  | string  | 创建时间，ISO8601                 |
| updated_at | 更新时间  | string  | 更新时间，ISO8601                 |

## Requirements


### Requirement: 账户列表查询
系统应支持账户列表的展示，支持按账户名称、类型、状态等字段搜索或筛选。

#### Scenario: 加载账户列表
- 当用户进入账户管理页面
- 则系统自动加载账户数据

#### Scenario: 输入关键词搜索
- 当用户在搜索框输入关键词并提交
- 则系统按关键词过滤账户列表，显示匹配结果


### Requirement: 新增/编辑账户
系统应支持通过表单新增账户和编辑账户信息，表单字段包括：名称、账户类型、负责人、备注、状态。

#### Scenario: 新建账户表单校验
- 当用户点击“新增账户”并填写表单后提交
- 则系统校验必填项（名称、账户类型），校验通过则保存新账户，失败则显示错误提示

#### Scenario: 编辑账户信息
- 当用户在账户列表点击“编辑”并修改信息后保存
- 则系统保存变更


### Requirement: 账户状态变更
系统应支持用户在账户列表中对账户进行停用和启用操作，被停用账户不可用于新账单。

#### Scenario: 停用账户
- 当用户在账户列表点击“停用”按钮
- 则系统将该账户状态设为disabled，后续新账单无法选择该账户

#### Scenario: 启用账户
- 当用户在账户列表点击“启用”按钮
- 则系统将该账户状态设为enabled，可正常选择


### Requirement: 查看账户详情
系统应支持查看账户详细信息，展示所有字段。

#### Scenario: 查看详情抽屉
- 当用户点击账户列表的“查看”按钮
- 则系统右侧弹出抽屉，展示该账户的全部信息

## Purpose
资金账户管理：支持账户的分页查询、搜索、新增、编辑、禁用、查看详情等基础操作。

## Fields
| 字段名       | 中文名     | 类型     | 约束/说明                         |
|------------|----------|---------|----------------------------------|
| id         | 账户ID    | int     | 主键，自增                        |
| name       | 账户名称  | string  | 必填，唯一，1-64 字符              |
| type       | 账户类型  | string  | 必填，枚举：银行账户/支付宝/微信/现金/其他 |
| balance    | 余额      | number  | 必填，单位：元，精度2位            |
| status     | 状态      | string  | 必填，枚举：enabled/disabled       |
| remark     | 备注      | string  | 可选，最长 128 字符                |
| created_at | 创建时间  | string  | 创建时间，ISO8601                  |
| updated_at | 更新时间  | string  | 更新时间，ISO8601                  |

## Requirements


### Requirement: 账户列表分页查询
系统 SHALL 支持账户列表的分页展示，默认每页 20 条，并支持按账户名称、类型等字段模糊搜索。

#### Scenario: 分页加载账户列表
- **WHEN** 用户进入账户管理页面
- **THEN** 系统自动加载第一页账户数据，显示分页控件，可切换页码加载对应数据

#### Scenario: 输入关键词搜索
- **WHEN** 用户在搜索框输入关键词并提交
- **THEN** 系统按关键词过滤账户列表，显示匹配结果


### Requirement: 新增/编辑账户
系统 SHALL 支持通过表单新增账户和编辑账户信息（如名称、类型、备注等），名称不可与其他账户重复。新增账户时必填项校验，名称唯一。

#### Scenario: 新建账户表单校验
- **WHEN** 用户点击“新增账户”并填写表单后提交
- **THEN** 系统校验必填项，校验通过则保存新账户，失败则显示错误提示

#### Scenario: 编辑账户信息
- **WHEN** 用户在账户列表点击“编辑”并修改信息后保存
- **THEN** 系统保存变更，名称字段唯一性校验

### Requirement: 账户状态变更
系统 SHALL 支持管理员在账户列表中对账户进行禁用和启用操作，被禁用账户不可用于新账单，启用后可正常选择。

#### Scenario: 禁用账户
- **WHEN** 用户在账户列表点击“禁用”按钮
- **THEN** 系统将该账户状态设为禁用，后续新账单无法选择该账户

#### Scenario: 启用账户
- **WHEN** 用户在账户列表点击“启用”按钮
- **THEN** 系统将该账户状态设为启用，可正常选择

### Requirement: 查看账户详情
系统 SHALL 支持查看账户详细信息，展示所有字段。

#### Scenario: 查看详情抽屉
- **WHEN** 用户点击账户列表的“查看”按钮
- **THEN** 系统右侧弹出抽屉，展示该账户的全部信息

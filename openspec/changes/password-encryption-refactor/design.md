# 密码加密逻辑重构 Design

## How

### 1. 前端
- 用户注册/添加时，前端生成12位随机密码
- 前端实现加密算法：base64（去=）→反序→sha256→base64（去=），无盐
- 注册/登录时，前端仅传密文到后端
- 前端与后端加密算法需严格一致，建议单元测试覆盖

### 2. 后端
- 用户表字段统一为 password，类型建议 VARCHAR(64) 或更大
- 新增/修改用户时，直接存储前端传来的密文
- 登录校验时，直接比对前端传来的密文与数据库密文
- 清理 hash_password、verify_password 等冗余加密/校验逻辑
- 保留最大明文长度校验（20位）

### 3. 数据库
- 字段名统一 password，类型 VARCHAR(64) 或更大
- 如有历史字段（如 hashed_password），需迁移数据

### 4. 兼容性与安全性
- 明文密码最大20位，密文长度由算法自然决定
- 无盐加密，安全性有限，后续可考虑升级
- 所有传输需基于 HTTPS

### 5. 测试
- 前后端加密一致性测试
- 注册、登录、密码修改等流程全链路测试
- 历史数据迁移测试（如有）

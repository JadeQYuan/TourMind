## Context

本次变更为纯前端 UI 调整，不涉及后端 API 或数据模型变更。所有修改均集中在 Vue 组件内的文案字符串与 CSS/展开状态初始值。

当前状态：
- 行程列表缺少对未开始行程的撤销操作，用户无法直接在列表中撤销未出行行程
- 行程列表目的地字段旁显示状态标签，视觉噪音过多
- 行程表单每日明细默认折叠，需要额外点击才能操作
- 新增/编辑表单（行程、合同）底部包含提示语，用户认为冗余
- 合同列表缺少"编辑"快捷操作，需要进入详情页再点编辑
- 合同"复制链接"文案较技术化，改为"分享"更贴近用户语言

## Goals / Non-Goals

**Goals:**
- 统一前端文案与业务用语
- 减少用户操作步骤（行程明细默认展开、合同列表直接编辑）
- 去除不必要的 UI 噪音（目的地后状态标签、新建后提示弹窗）

**Non-Goals:**
- 不修改后端 API、数据库模型
- 添加的 `not_started → cancelled` 状态流转若后端已支持，前端直接调用即可；不新增迁移脚本
- 不重构组件结构

## Decisions

**决策 1：行程明细默认展开采用响应式初始值**
- 每日明细区块通常使用 `<a-collapse>` 或自定义展开控制，将 `activeKey`/展开状态的默认值改为"全部展开"即可
- 无需改动模板结构

**决策 2：合同列表"编辑"按钮仅对 `pending_sign` 显示**
- 与现有"复制链接"按钮的可见性逻辑一致，已签署和已撤销合同不允许编辑

**决策 3：行程"撤销"按钮仅对 `not_started` 状态显示**
- 未开始行程允许直接撤销；进行中/已完成的行程不能通过此按钮撤销
- 前端按 `status === 'not_started'` 控制按钮可见性

**决策 4：表单底部提示语直接删除，不替换**
- 提示语对有经验用户无信息价值，直接移除而非折叠或弱化

## Risks / Trade-offs

- [风险] 行程明细默认展开在数据条目多时页面较长 → 已有滚动容器，可接受
- [风险] 文案变更需同步检查 Mock 注释和测试数据 → 本次 Mock 无结构变更，风险极低

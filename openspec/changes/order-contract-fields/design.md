## Context

当前订单和合同管理模块字段不全，信息传递依赖手工录入，易出错且效率低。随着业务需求升级，需补充订单人数，合同需自动带入订单关键信息，合同列表字段需优化，提升数据一致性和用户体验。

## Goals / Non-Goals

**Goals:**
- 订单表新增人数字段，支持录入、展示、查询
- 合同新建/编辑时自动带入订单出行日期、天数、人数、定金、定金到账日期、尾款及到账日期，仅展示不可编辑
- 合同列表字段结构优化，增加甲方、联系方式、乙方、订单编号，去掉客户列

**Non-Goals:**
- 不涉及订单、合同以外的业务模块
- 不涉及合同字段的编辑权限调整（仅展示）

## Decisions

- 订单表新增“人数”字段，数据库、后端 schema、前端类型、页面均需同步
- 合同新建/编辑页面通过订单编号自动带入关键信息，前端只读展示，后端校验
- 合同列表字段结构调整，前后端、Mock、导出等均需同步
- 采用 Alembic 生成数据库迁移脚本，确保数据平滑升级
- 前端类型定义、API、Mock 数据需同步调整

## Risks / Trade-offs

- [风险] 现有合同列表字段变更影响导出/统计 → [缓解] 明确字段映射，充分测试
- [风险] 自动带入逻辑依赖订单数据完整性 → [缓解] 新建合同时校验订单必填字段
- [风险] 旧数据兼容性 → [缓解] 数据迁移脚本补齐缺失字段

## Migration Plan
1. 订单表新增人数字段，生成并执行数据库迁移脚本
2. 后端 schema、API、前端类型、页面同步支持人数字段
3. 合同新建/编辑页面实现自动带入订单关键信息逻辑
4. 合同列表字段结构调整，前后端、Mock、导出等同步
5. 补充/更新测试用例，验证新功能和兼容性

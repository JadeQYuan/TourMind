## Context

当前系统各业务模块页面布局和响应式逻辑实现分散，缺乏统一标准，导致桌面端与移动端体验不一致，维护难度大。部分页面未严格区分主内容区、侧边栏、顶部导航等结构，Table/Card 双视图切换行为不统一。随着移动端访问量提升，需制定统一规范，提升用户体验和开发一致性。

## Goals / Non-Goals

**Goals:**
- 明确主内容区、侧边栏、顶部导航等页面结构标准
- 统一桌面端与移动端的响应式断点与布局切换逻辑
- 规范 Table/Card 双视图切换行为
- 统一 Drawer、Form、操作栏等通用组件布局
- 降低维护成本，提升开发效率

**Non-Goals:**
- 不涉及后端 API 或数据结构变更
- 不包含具体业务流程优化，仅聚焦前端布局与响应式

## Decisions

- 采用 Ant Design Vue 官方推荐的布局组件（Layout、Sider、Header、Content）为基础，统一主结构
- 响应式断点统一为 768px，桌面端为 Table 视图，移动端为 Card 视图，断点切换逻辑由全局 composable 实现
- 所有列表页均需实现 Table/Card 双视图，保持交互一致性
- Drawer 宽度桌面端 480px，移动端全屏，底部操作栏使用 sticky 定位
- 通用组件（如操作栏、表单）按新规范重构，避免重复实现
- 现有页面逐步迁移，优先核心业务模块

## Risks / Trade-offs

- [风险] 现有页面样式和交互可能受影响 → [缓解] 分阶段迁移，充分测试
- [风险] 部分自定义组件与新规范不兼容 → [缓解] 统一重构或适配
- [风险] 移动端兼容性问题 → [缓解] 多终端测试，及时修正

## Migration Plan

1. 制定并发布布局与响应式规范文档
2. 新开发页面全部按新规范实现
3. 现有页面分阶段迁移，优先订单、行程、合同等核心模块
4. 通用组件重构，替换旧实现
5. 多端测试，收集反馈，持续优化

## Open Questions

- 是否需要支持更多断点（如平板端）？
- 旧页面迁移是否需设定截止时间？
- 是否需提供主题/配色自定义能力？

## Context

当前合同管理模块存在字段不全、内容分散等问题，影响合同合规性和业务流转效率。需支持甲乙方及联系电话的完整录入与自动带入，合同查看页需结构化展示关键信息，提升用户体验和数据一致性。

## Goals / Non-Goals

**Goals:**
- 合同新增/编辑支持甲乙方及联系电话字段，甲方及电话自动从订单带入，支持编辑
- 合同查看页内容结构化，分区展示基本信息、款项、行程、补充信息
- 前后端数据结构、API、Mock、表单、详情视图同步调整

**Non-Goals:**
- 不涉及合同签署流程、PDF 生成、历史数据迁移
- 不调整订单、行程等其他业务模块核心流程

## Decisions

- 前端合同表单与详情页采用 Ant Design Vue 组件，Drawer 右侧抽屉统一承载
- 合同数据结构新增 party_a、party_a_phone、party_b、party_b_phone 字段，后端 models/schemas/api 均需同步
- 甲方及联系电话从订单接口获取默认值，前端表单支持编辑
- 合同详情页分区渲染，结构化展示基本信息、款项、行程、补充信息，行程明细通过订单接口获取
- Mock 层需支持新字段及结构化数据

## Risks / Trade-offs

- [兼容性] → 需确保旧合同数据展示不报错，必要时字段兜底处理
- [数据一致性] → 订单与合同数据同步需防止信息遗漏，接口需防御性校验
- [前端复杂度] → 详情页分区渲染需统一样式，避免 UI 不一致

## Migration Plan

1. 后端 models/schemas/routers 增加新字段，接口支持带入与编辑
2. 前端表单、详情页、API、Mock 同步调整
3. 行程明细通过订单接口获取，前端渲染
4. 测试新建、编辑、查看合同全流程，兼容旧数据
5. 部署上线，监控异常日志

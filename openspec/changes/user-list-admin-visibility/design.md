## Context

当前系统的用户管理模块允许管理员（admin）和系统管理员（system_admin）均可查看和添加所有用户，未做严格区分，存在权限越权风险。

## Goals / Non-Goals

**Goals:**
- 系统管理员可查看所有用户并添加任意角色用户
- 管理员仅可查看非系统管理员用户，并且只能添加管理员和助理
- 前后端权限校验一致，防止越权

**Non-Goals:**
- 不涉及其它业务模块权限调整
- 不涉及用户角色体系的新增/删除

## Decisions

- 后端接口（用户列表、用户新增）根据当前登录用户角色动态过滤可见用户和可选角色
- 前端页面根据当前用户角色动态渲染可见用户、可选角色
- 统一通过后端接口返回的角色列表驱动前端角色选择器，避免硬编码
- 所有角色判断以后端 JWT/Session 权限为准，前端仅做 UI 适配

## Risks / Trade-offs

- [风险] 前端仅做 UI 隐藏，未做后端校验 → [缓解] 必须后端接口强制校验
- [风险] 角色判断遗漏，导致部分用户越权 → [缓解] 增加单元测试和集成测试覆盖

## Migration Plan

- 后端接口调整后，前端同步适配
- 测试所有角色下的用户列表和新增用户功能
- 回滚方案：如有问题，恢复为原有无区分逻辑

## Context

当前合同管理模块在新增/编辑合同时，订单选择项未置于表单最上方，用户需先填写甲乙方等信息，后选订单，易导致数据不一致或遗漏。甲乙方及联系方式在 PC 端为多行展示，信息密度低，录入效率不高。

## Goals / Non-Goals

**Goals:**
- 合同表单新增/编辑时，订单选择项置于最顶部，强制先选订单
- PC 端甲乙方及联系方式同排展示，提升信息密度
- 保持移动端兼容性，表单样式自适应
- 相关 API、类型、Mock 数据同步调整

**Non-Goals:**
- 不涉及合同签署流程、状态流转逻辑
- 不调整订单、产品等其他业务模块表单结构

## Decisions

- 前端采用 Ant Design Vue 的 `<a-form-item>` + `<a-row>`/`<a-col>` 实现同排布局，PC 端 breakpoint ≥ 992px 时甲乙方信息同排，移动端自动换行
- 订单选择项使用 `<a-select>`，置于表单最上方，选中后自动带入订单相关信息
- 类型定义（ContractCreate/Update）同步调整，API 层参数校验
- Mock 层防御性处理 params，确保无订单 id 时返回错误
- 后端接口参数顺序、schema 校验同步调整

## Risks / Trade-offs

- [表单兼容性] → 需充分测试移动端/PC 响应式布局，防止样式错乱
- [数据一致性] → 订单选择后需自动刷新甲乙方信息，防止用户误操作
- [Mock 数据] → Mock 层需防御无订单 id 情况，避免前端假数据误导

# 智游管家 TourMind  产品需求文档（PRD）

> 版本：v2.0  日期：2026-04-17  状态：草稿

面向中小型旅行社或旅游从业者的轻量化一体化旅游业务管理平台，以**订单**为核心，串联行程规划、合同签署、财务核算的完整业务链路。

---

## 一、基础配置

#### 客户端签署流程（外部访问，无需登录）

客户通过分享链接访问合同页，依次完成以下步骤：

**步骤一：手机号验证**

- 页面展示合同摘要（客户姓名、产品名称、出行日期、总金额）
- 客户输入手机号，系统校验是否与订单预留手机号一致
  - 一致：进入下一步
  - 不一致：提示「手机号与预留信息不符，请联系旅行社」

**步骤二：阅读合同**

- 完整展示合同内容（行程明细、费用条款、退改规则等，内容来自订单和产品）
- 客户须滚动至底部方可激活「同意并签署」按钮

**步骤三：电子签名**

- 提供手写签名画板（触屏/鼠标均可）
- 客户完成签名后点击确认

**步骤四：上传证件信息**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 证件类型 | 枚举 | ✅ | 身份证 / 护照 / 港澳通行证 |
| 证件正面照 | 图片上传 | ✅ | JPG/PNG/WEBP，≤ 10 MB |
| 证件反面照 | 图片上传 | ❌ | 身份证必填反面 |
| 证件号码 | 文本 | ✅ | 手动输入，与上传证件核对 |

**步骤五：提交完成**

- 提交后合同状态自动流转至**已完成**
- 页面展示「签署完成」确认信息，显示合同编号和签署时间

## 四、用户认证与权限管理

### 4.1 登录

- 登录方式：**账号（手机号 / 工号）+ 密码**（内部系统，不接入第三方 OAuth）
- 登录页字段：账号（支持手机号、工号输入）、密码、「记住我」选项（勾选后 Token 有效期延长至 7 天，否则 8 小时）
- 失败提示统一为「账号或密码错误」，不区分哪个错（防止枚举）
- 登录成功后跳转至上次访问页，首次登录跳转首页（Dashboard）

### 4.2 会话管理

| 项目 | 规则 |
|------|------|
| 认证方式 | JWT（Bearer Token），存储于 `localStorage` |
| Token 有效期 | 普通：8 小时；记住我：7 天 |
| Token 刷新 | 过期前 30 分钟内有操作，自动静默刷新；过期后跳转登录页 |
| 多端登录 | 允许同账号多端同时登录（内部系统，无需踢下线） |
| 主动登出 | 点击「退出登录」，前端清除 Token，跳转登录页 |

### 4.3 密码规格

| 项目 | 规则 |
|------|------|
| 随机密码生成 | 前端生成，长度 12 位，包含大小写字母、数字、特殊字符 |
| 传输加密 | 前端使用 RSA 公钥对密码明文加密后传输，后端用私钥解密 |
| 存储加密 | 后端使用 bcrypt 对密码进行哈希存储，不保存明文 |
| 登录验证 | 前端对输入密码进行 RSA 加密后发送，后端解密后与 bcrypt 哈希比对 |

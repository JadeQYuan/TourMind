一、基础配置

1.1 产品管理
字段： 产品名称、目的地、天数、参考价格（起）、状态（上架/下架）、包含项目、不含项目、退改规则、出行须知、重要提示、备注
列表：产品名称、目的地、天数、参考价格（起）、状态（上架/下架）、创建时间
筛选：名称、目的地、状态
操作：新增、编辑、上架 / 下架，下架后不影响已关联的历史行程

行程模板：
字段： 序号、行程详情、住宿区域、相关凭证、备注

1.2 供应商管理
字段：名称、联系人、联系电话、状态（启用/停用）、备注
列表：名称、联系人、联系电话、状态（启用/停用）、创建时间
筛选：名称、状态
操作：新增、编辑、启用 / 禁用，禁用供应商后不可在新行程节点中选择，已关联记录不受影响


1.3 账户管理
字段：名称、类型、描述、用户、状态（启用/停用）
列表：名称、类型、描述、用户、状态（启用/停用）、创建时间
筛选：名称、类型、用户、状态
操作：新增、编辑、启用 / 禁用

1.4 用户管理
字段：姓名、手机号、工号、角色、状态（启用/禁用）、密码
列表：姓名、手机号、工号、角色、状态（启用/停用）、创建时间、最后登录时间
筛选：姓名、手机号、工号、角色、状态
操作：新增、编辑、启用 / 禁用、重置密码

角色：系统管理员，管理员，助理，其中系统管理员为初始化数据，姓名Admin，工号Admin，密码Admind0Tour
创建成功后生成随机密码，提示复制
重置密码时生成随机密码，提示复制


二、业务管理

2.1 订单管理

字段：订单编号、产品ID、客户姓名、手机号、出行日期、出行天数、价格、定金、供应商ID、成本、利润、状态
列表：产品、客户、手机号、日期、供应商、价格/成本/利润
筛选：产品、供应商、手机号、姓名
操作：新增、编辑

状态：待下定，待付款，已完成

2.2 行程管理

字段：订单ID、产品ID、人数、出行人员名单、开始日期、结束日期、状态
列表：订单编号、产品名称、出发地-目的地、客户、手机号、开始日期、结束日期、状态
筛选：订单编号、产品名称、手机号、姓名、状态
操作：新增、编辑、分享

行程明细：
字段： 出行日期、行程详情、住宿区域、相关凭证、备注

行程详情预览
- 以时间线形式展示完整行程
- 支持生成外部分享链接，客户无需登录即可通过链接查看行程详情（只读）
- 分享链接含唯一访问令牌，支持随时撤销


2.3 合同管理
字段：订单ID、状态、签名、相关证件
列表：订单编号、状态
筛选：订单编号、状态
操作：新增、分享、查看

状态：待签署、已完成

合同签署


2.4 账单管理
字段：订单ID、收入/支出、金额、备注、账户、附件、类型、时间
列表：订单编号、收入/支出、金额、账户、类型、时间
筛选：订单编号、收入/支出、账户、类型、时间
操作：收入、支出、编辑

收入类型： 定金 / 尾款 / 全款 / 其他 
支出类型： 供应商付款 / 运营成本 / 其他

四、用户认证与权限管理
3.1 登录

- 登录方式：**账号（手机号 / 工号）+ 密码**（内部系统，不接入第三方 OAuth）
- 登录页字段：账号（支持手机号、工号输入）、密码、「记住我」选项（勾选后 Token 有效期延长至 7 天，否则 8 小时）
- 登录失败处理：
  - 失败提示统一为「账号或密码错误」，不区分哪个错（防止枚举）
- 登录成功后跳转至上次访问页，首次登录跳转首页（Dashboard）

3.2 会话管理
认证方式 | JWT（Bearer Token），存储于 `localStorage` |
| 多端登录 | 允许同账号多端同时登录（内部系统，无需踢下线） |
| 主动登出 | 点击「退出登录」，前端清除 Token，跳转登录页（后端无需维护黑名单） |
